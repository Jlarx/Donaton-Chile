
POSTMAN (DONATON)

Nota: Todas las peticiones deben hacerse a través de 
la CENTRAL (Gateway) que corre en el puerto 8080.


1. LOGÍSTICA (Centros de Acopio)

IMPORTANTE: Ahora que el sistema está asegurado, debes enviar el token 
JWT en la pestaña "Headers" (o "Authorization" -> Bearer Token) en 
todas las peticiones de Logística, Donaciones y Necesidades.
Key: Authorization
Value: Bearer <tu_token>

0. AUTENTICACIÓN (Usuarios)

Nota: Puedes usar cualquier correo electrónico que desees. El rol se define en el campo "rol".

Registrar un nuevo Administrador (ADMIN)
URL: http://localhost:8080/api/auth/registro
Método: POST

{
  "email": "coordinador@donaton.com",
  "password": "123",
  "nombre": "Admin General",
  "rol": "ADMIN"
}

Registrar un nuevo Voluntario (USER)
URL: http://localhost:8080/api/auth/registro
Método: POST

{
  "email": "juan@voluntario.com",
  "password": "123",
  "nombre": "Juan Pérez",
  "rol": "USER"
}

Iniciar Sesión (Login - Obtener JWT)
URL: http://localhost:8080/api/auth/login
Método: POST

{
  "email": "coordinador@donaton.com",
  "password": "123"
}
*Respuesta esperada: {"token": "eyJhbGciOiJIUzI1NiJ9..."}*


1. LOGÍSTICA (Centros de Acopio)

Crear un Centro de Acopio (Requiere Token de ADMIN)
URL: http://localhost:8080/api/logistica/centros
Método: POST

{
  "nombre": "Centro Base Lima",
  "ubicacion": "Av. Principal 123, Lima",
  "capacidadMaxima": 500,
  "inventarioActual": 0
}

Listar todos los Centros de Acopio (Requiere Token)
URL: http://localhost:8080/api/logistica/centros
Método: GET

Eliminar un Centro de Acopio (Requiere Token de ADMIN)
URL: http://localhost:8080/api/logistica/centros/1
Método: DELETE

Asignar Recursos Masivos - Strategy (Requiere Token de ADMIN)
URL: http://localhost:8080/api/logistica/asignaciones?cantidad=100
Método: POST
*Respuesta esperada: Mensaje confirmando la repartición.*

2. DONACIONES

Listar todas las Donaciones (Requiere Token)
URL: http://localhost:8080/api/donaciones
Método: GET

Eliminar una Donación (Requiere Token de ADMIN)
URL: http://localhost:8080/api/donaciones/1
Método: DELETE

3. NECESIDADES

Registrar una nueva Necesidad (Requiere Token de ADMIN)
URL: http://localhost:8080/api/necesidades
Método: POST

{
  "titulo": "Falta Agua",
  "descripcion": "Agua potable para 50 familias",
  "tipoRecurso": "ALIMENTO",
  "cantidadRequerida": 100,
  "ubicacion": "Zona Sur"
}

Listar todas las Necesidades (Requiere Token)
URL: http://localhost:8080/api/necesidades
Método: GET

Eliminar una Necesidad (Requiere Token de ADMIN)
URL: http://localhost:8080/api/necesidades/1
Método: DELETE

4. PRUEBAS DE VALIDACIÓN Y ERRORES PERSONALIZADOS

Intentar crear un Centro de Acopio SIN Datos (Falla la Validación)
URL: http://localhost:8080/api/logistica/centros
Método: POST

{
  "nombre": "",
  "ubicacion": "",
  "capacidadMaxima": 0
}
*Respuesta esperada (HTTP 400 Bad Request): Un JSON con los mensajes de validación personalizados indicando que el nombre es obligatorio y la capacidad debe ser mayor a 0.*

Intentar registrar una Donación Inválida (Falla la Validación)
URL: http://localhost:8080/api/donaciones
Método: POST

{
  "tipo": "",
  "recurso": "",
  "cantidad": 0,
  "origen": "",
  "centroAcopioId": null
}


5. PRUEBA DE CIRCUIT BREAKER (FALLBACK)

Para probar que el sistema de tolerancia a fallos funciona:
1. Apaga/Detén el microservicio de "Donaciones" en VS Code.
2. Ejecuta en Postman el GET: http://localhost:8080/api/donaciones
3. En lugar de un error feo, el Gateway interceptará la 
   falla y te devolverá el mensaje del Fallback:
   "El servicio de donaciones no está disponible actualmente..."


Para correr el programa:

cd .\productos\donaton\ 

.\start-all.bat
